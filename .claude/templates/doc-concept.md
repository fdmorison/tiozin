# Doc: Concept Page

## Overview

A concept page explains one or more Tiozin concepts, their purpose, and how they fit into the system.

## Rules

- Explain what the concept is and why it exists.
- Explain how the concept relates to other concepts.
- Explain behavior and design decisions, not implementation details.
- Include examples only when they help clarify the concept.
- Use tables for variants, classifications, relationships, or comparisons when appropriate.
- Verify every code example against source code or tests before publishing.
- Link to reference pages or conceptual pages instead of documenting them again.

## Template

```markdown
# <Title>

[brief description]

[concept explanation]

## <Subject>

[content]

[example if needed]

## <Subject>

[content]

[example if needed]

## Related

- How to ...
- Related component
- Related concept
```
